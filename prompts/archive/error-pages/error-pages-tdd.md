## Architecture Overview

There are two independent error-handling layers, each solved separately:

```
Browser request
      │
      ▼
   Nginx (port 80)
      │
      ├── /api/**  ──────────────────► Spring Boot :8080
      │                                      │
      ├── /[shortCode]  ────────────► Spring Boot :8080
      │                                      │
      │                               CustomErrorController
      │                               (/error mapping)
      │                                      │
      │                               JSON ──┤── HTML page
      │
      └── everything else  ──────────► React SPA (Vite/Nginx :80)
                                             │
                                       React Router
                                             │
                                    path="*" ─► <NotFound />
                                    /link-expired ─► <LinkExpired />
```

## Backend: `CustomErrorController`

### Location

```
backend/src/main/java/com/avivly/urlshortener/controller/CustomErrorController.java
```

### Approach

Spring Boot forwards any unhandled error (no matching handler, exception, etc.) to `/error`. Implementing `ErrorController` claims that mapping, suppressing the Whitelabel fallback.

### Class Signature

```java
@Controller
public class CustomErrorController implements ErrorController {

    @RequestMapping("/error")
    public ResponseEntity<?> handleError(HttpServletRequest request) { ... }
}
```

### Request-Type Detection

```java
String accept = request.getHeader("Accept");
String uri = String.valueOf(request.getAttribute(RequestDispatcher.ERROR_REQUEST_URI));

boolean jsonRequest =
    (accept != null && accept.contains(MediaType.APPLICATION_JSON_VALUE))
    || uri.startsWith("/api/");
```

### Status Code Resolution

```java
Object statusAttr = request.getAttribute(RequestDispatcher.ERROR_STATUS_CODE);
int status = statusAttr != null ? Integer.parseInt(statusAttr.toString()) : 500;
```

### JSON Response (API path)

```java
return ResponseEntity.status(status).body(Map.of(
    "status",  status,
    "error",   httpStatus.getReasonPhrase(),
    "message", descriptionFor(status)
));
```

Response shape is identical to `GlobalExceptionHandler` to keep the API contract consistent.

### HTML Response (browser path)

```java
return ResponseEntity.status(status)
    .contentType(MediaType.TEXT_HTML)
    .body(htmlPage(status, statusText, description));
```

The HTML is generated by a private `htmlPage(int, String, String)` method returning a `String` via a text block. No template engine is required.

### Error Message Map

| Status | Message |
|--------|---------|
| 400 | The request was invalid or malformed. |
| 403 | You don't have permission to access this resource. |
| 404 | The page you're looking for doesn't exist. |
| 405 | The HTTP method is not allowed for this endpoint. |
| 500 | An unexpected error occurred on our end. |
| 503 | The service is temporarily unavailable. |
| default | Something went wrong. Please try again later. |

### HTML Page Design Spec

**Structure**

```
<body> (flex column, min-height 100vh, bg #f9fafb)
  <header> (bg #2563eb, text white, 1rem/1.5rem padding, box-shadow)
    <h1> "URL Shortener"
  <main> (flex-1, items-center, justify-center)
    <div class="card"> (bg white, rounded .75rem, shadow, max-width 420px)
      <div class="icon-wrap"> (inline-flex, w-16 h-16, rounded-full, bg #eff6ff)
        <svg>  {statusIcon}  — Heroicons outline, 32×32, stroke #2563eb, strokeWidth 1.5
      <div class="code">  {status}       — 5rem, weight 800, color #2563eb
      <p class="title">   {statusText}   — 1.2rem, weight 600, color #111827
      <p class="desc">    {description}  — 0.9rem, color #6b7280
      <a href="/" class="btn"> "Back to Dashboard"
  <footer> (text-center, 0.75rem, color #9ca3af)
    "Avivly URL Shortener"
```

**Per-status icons (Heroicons outline, inline SVG)**

| Status | Icon |
|--------|------|
| 400 | `x-circle` |
| 403 | `lock-closed` |
| 404 | `magnifying-glass` |
| 405 | `no-symbol` |
| 500 | `wrench-screwdriver` |
| 503 | `clock` |
| default | `exclamation-circle` |

Icons are resolved by a private `iconFor(int status)` method returning an SVG path string. Each is rendered inside a `rounded-full` container with a light blue background (`#eff6ff`), consistent with the `LinkExpired` icon circle pattern. No external icon library — all SVGs are inlined as text-block strings.

**Colour tokens (inline CSS)**

| Token | Hex |
|-------|-----|
| Page background | `#f9fafb` |
| Header | `#2563eb` |
| Card border | `#e5e7eb` |
| Icon circle bg | `#eff6ff` |
| Icon stroke | `#2563eb` |
| Status code | `#2563eb` |
| Title | `#111827` |
| Description | `#6b7280` |
| CTA button | `#2563eb` → hover `#1d4ed8` |
| Footer | `#9ca3af` |

**Responsive rule** — single media query at `max-width: 480px`:
- Card padding reduces to `2rem 1.5rem`
- Status code font-size reduces to `3.5rem`

**Constraints**
- Inline CSS only — no `<link>`, `<script>`, or external resources.
- Target page weight < 3 KB.
- No stack traces or internal error details in the response body.

## Frontend: `NotFound` Component

### Location

```
frontend/src/components/NotFound.jsx
```

### Structure

```jsx
import { Link } from 'react-router-dom';

export default function NotFound() {
  return (
    <div className="min-h-screen bg-gray-50 flex flex-col">
      <header>…</header>          {/* same as App.jsx header */}
      <main className="flex-1 flex items-center justify-center …">
        <div className="bg-white rounded-xl shadow p-10 text-center max-w-md w-full">
          <p className="text-8xl font-extrabold text-blue-600 …">404</p>
          <h2 …>Page Not Found</h2>
          <p …>The page you're looking for doesn't exist or has been moved.</p>
          <Link to="/" …>Back to Dashboard</Link>
        </div>
      </main>
    </div>
  );
}
```

### Route Registration

In `App.jsx`, add a wildcard catch-all as the last `<Route>`:

```jsx
import NotFound from './components/NotFound';

// inside <Routes>:
<Route path="*" element={<NotFound />} />
```

The wildcard must be the **last** route to avoid shadowing `/link-expired`.

## Frontend: `LinkExpired` Component (redesign)

### Location

```
frontend/src/components/LinkExpired.jsx   (existing file, full rewrite)
```

### Structure

```jsx
import { Link } from 'react-router-dom';

export default function LinkExpired() {
  return (
    <div className="min-h-screen bg-gray-50 flex flex-col">
      <header>…</header>
      <main className="flex-1 flex items-center justify-center …">
        <div className="bg-white rounded-xl shadow p-10 text-center max-w-md w-full">
          {/* Eye-slash SVG icon, 64×64, red-50 bg circle */}
          <div className="inline-flex items-center justify-center w-16 h-16 rounded-full bg-red-50 mb-4">
            <svg …/>   {/* eye-slash, stroke red-500, strokeWidth 2 */}
          </div>
          <h2 …>Link Unavailable</h2>
          <p …>This link has expired, reached its click limit, or no longer exists.</p>
          <Link to="/" …>Back to Dashboard</Link>
        </div>
      </main>
    </div>
  );
}
```

**Icon choice:** Heroicons `eye-slash` outline (inline SVG, no dependency). Rendered inside a `rounded-full bg-red-50` container; stroke colour `text-red-500`.

### Unchanged Behaviour

- Route `/link-expired` remains in `App.jsx`.
- `RedirectController` continues to redirect invalid/expired codes to `/link-expired`.

## Shared Component Pattern (header)

Both `NotFound` and `LinkExpired` use the same header block as `App.jsx`:

```jsx
<header className="bg-blue-600 text-white px-6 py-4 shadow">
  <h1 className="text-2xl font-bold">URL Shortener</h1>
</header>
```

No shared component abstraction is introduced to keep the change minimal. If a third error surface is added later, extracting a shared `AppHeader` component is appropriate at that point.
